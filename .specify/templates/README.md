## Usage

Templates are used by agents automatically. You can also manually initialize a feature:

```bash
# Initialize new feature from templates
.specify/scripts/new-feature.sh "feature-name"
# Preferred CLI entry
sdd new feature-name --template standard
```

Current scaffold behavior also creates compatibility artifacts:
- `design.md` (design alias alongside `plan.md`)
- `implementation.md` (implementation tracking skeleton)

These two files are generated by the scaffold scripts, not by standalone template files.

## Placeholder Conventions

- `[FEATURE_NAME]` - Replace with actual feature name
- `[NNN]` - Feature number (e.g., 001, 042)
- `[feature-slug]` - URL-safe feature identifier
- `[DATE]` - Current date in YYYY-MM-DD format
- `<!-- INSTRUCTION -->` - Guidance for the agent (remove in final)

## Customization

To customize templates for your project:

1. Copy the template you want to modify
2. Edit to match your conventions
3. Update this README if adding new templates

Templates should be kept minimal - add project-specific details to the constitution instead.
